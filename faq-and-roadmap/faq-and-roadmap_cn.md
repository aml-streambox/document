# 常见问题与路线图

## 已知问题

### 系统稳定性

当前系统稳定性有限，可能出现以下问题：

- **绿屏** - 视频管线错误，通常需要重启
- **黑屏** - HDMI 信号或管线异常，可能恢复或需重启
- **系统崩溃** - 内核或驱动崩溃，需要重启

以下场景更易触发上述问题：
- HDMI 热插拔（线缆连接/断开）
- 分辨率或帧率变更
- 长时间运行

**解决方法**：出现问题时重启设备。系统设计为启动后自动启动管线。

### HDR 编码限制

硬件编码器支持 10-bit HDR 编码，但受固件限制，VDIN1 和 v4l2src 目前暂不支持 10-bit HDR 采集。HDR 透传功能正常，HDR 直播将在后续版本实现。

### HEVC 无损编码缺陷

硬件支持无损 HEVC 编码，但此功能目前存在缺陷且无法启用。CBR 和 VBR 模式可正常工作。

### 1080p240Hz 支持

硬件支持 1080p240Hz，但 VIC（视频识别码）定义暂不支持该时序。目前仅支持 1080p120Hz。

## 常见问题

### Q：为何必须连接 HDMI TX 才能进行采集？

**A：** 默认配置下，GStreamer Manager 插件采集 HDMI TX 输出而非直接采集 HDMI RX。管线流程为：

```
HDMI RX → tvserver → HDMI TX → GStreamer (v4l2src) → 编码器 → 网络
```

此设计在实现超低延迟透传的同时，采集相同视频用于直播。如未连接 HDMI TX，视频管线不会产生采集输出。

### Q：系统是否支持 HDR 透传？

**A：** 支持，最高 4K60fps HDR 透传。HDR 元数据正确从 HDMI RX 传递到 HDMI TX。

### Q：是否支持 4K60fps 直播？

**A：** 支持 4K60fps 编码与直播。编码器从 500MHz 超频至 666MHz 以支持此功能。

### Q：支持哪些直播协议？

**A：** 支持的协议取决于 GStreamer 的支持范围。开发者主要使用和测试 **SRT** 协议，推荐在不稳定的网络环境中使用以获得更好稳定性。

**RTMP**、**NDI** 等其他协议应可通过 GStreamer 插件支持，但尚未经过充分测试。可使用 GStreamer Manager 中的手动管线编辑器配置自定义协议。

### Q：是否支持 HDCP？

**A：** A311D2 硬件支持 HDCP，但因法律及许可限制，本项目不计划实现 HDCP 支持。

### Q：是否支持杜比视界/全景声？

**A：** 硬件具备杜比视界和杜比全景声处理能力，但因法律及许可限制，不计划实现对这些格式的支持。

### Q：如何访问 Web 界面？

**A：** 通过 WiFi 热点连接设备（默认地址：`http://192.168.172.1:9090`）或通过以太网使用 DHCP 分配的 IP 地址访问。

### Q：能否使用自定义 GStreamer 管线？

**A：** 可以，Cockpit GStreamer Manager 插件支持手动管线编辑，还提供 AI 助手帮助将自然语言描述转换为管线配置。

### Q：为何我的源没有音频？

**A：** 确保 HDMI 源输出 PCM 音频。某些压缩音频格式可能无法正确处理。还要验证音频环回设备是否正确配置。

## 开发计划

- **HDR 直播** - 固件支持可用后启用 10-bit HDR 采集与直播
- **稳定性改进** - 改进 HDMI 热插拔处理与管线恢复
- **缺陷修复** - 修复 HEVC 无损编码问题

- **1080p240Hz 支持** - 为 1080p240Hz 时序添加 VIC 定义
- **音频增强** - 改进压缩音频格式支持

## 问题反馈

如遇到上述未列出的问题：

1. 查阅上方[已知问题](#已知问题)章节
2. 重启设备后重试
3. 在 https://github.com/aml-streambox/yocto/issues 提交 Issue

提交时请提供：
- 设备型号（VIM4、TVPro 等）
- 输入源（PS5、Xbox、PC 等）
- 分辨率与帧率
- 重现步骤
